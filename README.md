# Tennis Management System & SQL Portfolio

This repository showcases my expertise in **Relational Database Design** and **SQL Programming**. It features a primary sports management project that evolved through different architectural requirements, along with a collection of specialized SQL exercises.

## 1. Tennis Pro Management System (Core & Extensions)
The main project is a modular database designed to manage professional tennis ecosystems. It demonstrates my ability to handle scalability and complex relationships.

* **01-Core-Model:** Foundational schema centered on the `Tenistas` (Tennis Players) entity, managing player profiles, nationalities, and core statistics.
* **02-Trainers-Extension:** Integrated coaching staff management, handling many-to-many relationships between athletes and technical teams.
* **03-Courts-Extension:** Added infrastructure for `Pistas` (Courts) and venues, focusing on surface types (clay, grass, hard) and scheduling.
* **04-Rankings-Extension:** Implemented historical point systems and ATP/WTA ranking scales to track player performance over time.

## 2. SQL Problem Solving (05-Other-Exercises)
A collection of additional SQL scripts and challenges focusing on:
* Complex **JOIN** operations across multiple tables.
* Data aggregation and reporting using **GROUP BY** and **HAVING**.
* Subqueries and CTEs (Common Table Expressions) for advanced data filtering.
* Database integrity through **Constraints** and **Triggers**.

## Technical Skills Demonstrated
* **Schema Design:** Entity-Relationship (ER) modeling and normalization.
* **DML/DDL:** Proficiency in writing clean, executable `schema.sql` and `populate.sql` scripts.
* **Integrity:** Handling Foreign Keys, Unique constraints, and data types.
